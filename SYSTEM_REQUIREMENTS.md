# Системные требования для торгового советника по опционам

**Операционная система:** Ubuntu 22.04 (без графического интерфейса)  
**Количество опционов в подписке:** 80

## Оценка ресурсов

### Архитектура приложения

Приложение состоит из 3 сервисов в Docker:
1. **telegram-bot** - Telegram бот для взаимодействия с пользователем
2. **api-server** - FastAPI сервер для данных опционов (порт 8000)
3. **monitoring-service** - Сервис мониторинга (порт 8001)

Дополнительные компоненты:
- WebSocket подключение к Bybit для 80 опционов
- In-memory хранилище (data_store) для текущих данных
- SQLite БД для истории опционов (сохранение каждые 5 минут, только OTM опционы)
- LLM агент (DeepSeek API) - запускается каждый час

---

## Оперативная память (RAM)

### Детальный расчет:

1. **Python runtime (3 процесса):**
   - Базовый Python 3.12 процесс: ~30-50 MB каждый
   - Итого: **90-150 MB**

2. **In-memory data_store (80 опционов):**
   - Данные по каждому опциону: ~500 bytes
   - Метаданные и структуры: ~1-2 MB
   - Итого: **~2 MB**

3. **SQLite БД (история опционов):**
   - Сохранение: каждые 5 минут = 288 раз в день
   - Фильтрация: только OTM опционы (~50% = 40 опционов)
   - Записи в день: 40 опционов × 288 = ~11,520 записей
   - Размер записи: ~200 bytes (с индексами)
   - За месяц: ~70 MB данных + индексы = **~100-150 MB**
   - БД находится на диске, но SQLite кэширует часть в RAM: **~20-30 MB**

4. **WebSocket буферы (Bybit):**
   - 80 подписок, обновления каждые 1-5 секунд
   - Буферы: **~1-2 MB**

5. **Telegram bot библиотека:**
   - python-telegram-bot: **~20-30 MB**

6. **FastAPI серверы (2 сервиса):**
   - Uvicorn + FastAPI: ~20-30 MB каждый
   - Итого: **~40-60 MB**

7. **Дополнительные библиотеки:**
   - pandas, numpy (анализ данных): **~30-50 MB**
   - APScheduler (планировщик): **~5 MB**

8. **LLM агент:**
   - HTTP клиент для DeepSeek API: **~5-10 MB**
   - Обработка ответов в памяти (во время работы): **~10-20 MB**

9. **Docker overhead:**
   - Каждый контейнер: ~10-20 MB
   - Итого для 3 контейнеров: **~30-60 MB**

10. **Резерв для пиков и системных процессов:**
    - 20-30% резерв: **~100-150 MB**

### Итого оперативная память:

- **Минимальные требования:** **512 MB RAM**
- **Рекомендуемые требования:** **1 GB RAM** (с комфортным запасом)

**Примечание:** При работе с историей более 1-2 месяцев размер БД может вырасти до 200-300 MB на диске, но в RAM будет кэшироваться только активная часть.

## Процессор (CPU)

### Детальный расчет:

1. **WebSocket обработка:**
   - 80 опционов, обновления каждые 1-5 секунд
   - Парсинг JSON, обработка сообщений: **~1-2% одного ядра**

2. **Сохранение в БД (каждые 5 минут):**
   - Запись ~40 OTM опционов в SQLite
   - Время: ~1-2 секунды, нагрузка: **~5-10% одного ядра** (периодически)

3. **LLM агент (каждый час):**
   - HTTP запрос к DeepSeek API: ~5-15 секунд (в основном I/O ожидание)
   - Обработка ответа: ~1-2 секунды
   - Пиковая нагрузка: **~20-30% одного ядра** во время работы
   - Средняя нагрузка: **~0.5-1% одного ядра**

4. **Telegram bot:**
   - Обработка команд и сообщений: **~1-2% одного ядра**

5. **FastAPI серверы:**
   - Минимальная нагрузка: **~1-2% одного ядра** каждый

6. **Анализ данных (IVR, греки):**
   - При запуске агента: обработка истории из БД
   - Пиковая нагрузка: **~10-15% одного ядра** на 5-10 секунд

### Итого процессор:

- **Минимальные требования:** **1 ядро** (1000-1500 MHz, современная архитектура)
- **Рекомендуемые требования:** **1-2 ядра** (для комфортной работы и пиковых нагрузок)

**Примечание:** Приложение не требует высокой вычислительной мощности. Основная нагрузка - это I/O операции (WebSocket, HTTP запросы, запись в БД).

## Дисковое пространство

### Детальный расчет:

1. **SQLite БД (история опционов):**
   - За месяц: ~100-150 MB (только OTM опционы)
   - За 3 месяца: ~300-450 MB
   - За год: ~1.2-1.8 GB
   - Рекомендуется хранить историю **3-6 месяцев** = **~300-900 MB**

2. **Python окружение:**
   - Зависимости (pandas, numpy, fastapi, telegram-bot и др.): **~200-300 MB**

3. **Docker образы:**
   - Базовый образ Python 3.12-slim: ~150-200 MB
   - Финальный образ с зависимостями: **~400-600 MB**
   - Всего для 3 сервисов (общий базовый образ): **~500 MB - 1 GB**

4. **Логи:**
   - При нормальной работе: **~10-50 MB в месяц**
   - При отладке/ошибках: **~50-200 MB в месяц**

5. **Система (Ubuntu 22.04 минимальная установка):**
   - **~2-3 GB** (базовая установка без GUI)

6. **Резерв для обновлений и временных файлов:**
   - **~500 MB - 1 GB**

### Итого дисковое пространство:

- **Минимальные требования:** **5 GB** (с историей за 1 месяц)
- **Рекомендуемые требования:** **10 GB** (с историей за 3-6 месяцев и запасом)

## Сетевые требования

1. **Входящий трафик:**
   - WebSocket от Bybit: ~80 опционов × ~100 bytes каждые 1-5 сек = **~2-5 KB/сек** = **~7-18 MB/час**
   - Telegram обновления: **~1-5 MB/час** (зависит от активности)
   - Итого: **~10-25 MB/час** = **~240-600 MB/день**

2. **Исходящий трафик:**
   - HTTP запросы к DeepSeek API (каждый час): **~10-50 KB/запрос** = **~240 KB - 1.2 MB/день**
   - Telegram уведомления: **~1-10 MB/день** (зависит от количества сигналов)
   - Итого: **~2-15 MB/день**

### Итого сетевой трафик:

- **Минимальная пропускная способность:** **1 Mbps** (более чем достаточно)
- **Рекомендуемая пропускная способность:** **5-10 Mbps** (для стабильной работы)

## Оптимизация ресурсов

Если нужно уменьшить потребление ресурсов:

1. **Сократить историю БД:**
   - Удалять данные старше 1 месяца (вместо 3-6)
   - Экономия: ~100-150 MB на диске

2. **Увеличить интервал сохранения:**
   - Сохранять каждые 10 минут вместо 5 (изменить `DATA_CONFIG["save_interval_minutes"]`)
   - Экономия: ~50% места в БД

3. **Ограничить количество опционов:**
   - Уменьшить `strike_steps_count` в `SUBSCRIPTION_CONFIG` (например, с 7 до 5)
   - Экономия: пропорционально уменьшение нагрузки

4. **Отключить LLM агент:**
   - Если не используется, можно остановить сервис мониторинга
   - Экономия: ~30-50 MB RAM

5. **Использовать менее частый запуск агента:**
   - Изменить `AGENT_CONFIG["run_interval_minutes"]` с 60 до 120 минут
   - Экономия: ~50% запросов к DeepSeek API

Для мониторинга можно использовать:

```bash

docker compose logs --tail=100 telegram-bot
```

## Примечания

1. **SQLite БД:** Размер БД растет со временем. Рекомендуется настроить ротацию логов и периодическую очистку старых данных (например, через cron).

2. **Docker:** Все сервисы запускаются в отдельных контейнерах, но могут использовать общий базовый образ для экономии места.

3. **Ubuntu без GUI:** Экономит ~1-2 GB RAM и ~2-3 GB на диске по сравнению с установкой с GUI.

4. **SSD vs HDD:** Рекомендуется SSD для быстрой работы SQLite БД, особенно при росте истории.

5. **Стабильность сети:** Критична для работы WebSocket подключения к Bybit. При нестабильном соединении возможны переподключения и потеря данных.
